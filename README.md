# SMART SEGMENTATOR

## Описание
Система, которая с помощью машинного обучения автоматически разделяет клиентов на кластеры на основе анализа их личных данных. Она показывает эти группы в виде графиков и подробных профилей, что помогает принимать решения в маркетинге и управлении клиентами.

## Основные функции
- Кластеризация клиентов с подбором параметров и переменных;
- Интерактивное представление сегментов и анализ результатов;
- Хранение исходных данных и результатов кластеризации;
- Мониторинг качества класеризации.

## Установка и запуск
1. Установите зависимости: `pip install -r requirements.txt`
2. Настройте конфигурацию в `config/settings.yaml`
3. Запустите сервер: `python backend/main.py`
4. Запустите GUI: `python gui/main_window.py`

## Использование
1. Запустите приложение и на вкладке “Административная панель” выберите необходимые переменные и параметры для кластеризации;
2. Теперь вы можете построить график для отображения метода “Локтя” и выбрать оптимальное количество кластеров;
3. Теперь вы можете посмотреть на визуализацию сегментов;
4. На вкладке “Профили клиентов” вы можете посмотреть информацию о каждом профиле и изменить их описание при необходимости;
5. На вкладке “Анализа данных” вы можете посмотреть интерактивные диаграммы на основе данных о клиентах и результатов кластеризации.

## Структура проекта
- `data_analyzer.py` - модуль графического интерфейса PyQt6, экран анализа данных;
- `profiles_screen.py` - модуль графического интерфейса PyQt6, экран профилей клиентов;
- `admin_screen.py` - модуль графического интерфейса PyQt6, экран административной панели;
- `main.py` - модуль графического интерфейса PyQt6, точка входа для выполнения программы;
- `ml_models.py` - ML-модели и бизнес-логика;
- `/data` - демонстрационные данные и конфигурация;
- `/docs` - документация проекта.

## Связь реализованных функций с этапами системного анализа 
### Связь с Практикой 1 (Vision and Scope):
- GUI реализует взаимодействие с определенными стейкхолдерами (Пользователь, Администратор)
- Интерфейс отражает системные свойства: адаптивность, открытость, самообучаемость
### Связь с Практикой 2 (Классификация):
- Стохастичность системы отражена в том, что результаты кластеризации зависят от начальных условий
- Сложность архитектуры требует модульного GUI с четкой иерархией
- Самообучаемость реализована через подстройку параметров кластеризации и самообучение на новых данных
### Связь с Практикой 3 (Функциональное описание):
- Экраны GUI соответствуют функциональным блокам IDEF0
- Потоки данных между GUI и бэкендом реализуют DFD-диаграммы
- Морфология компонентов отражена в архитектуре интерфейса
### Связь с Практикой 4 (Динамическое моделирование):
- Состояния интерфейса синхронизированы с системным состоянием S(t)
- Пользовательские действия являются входными воздействиями U(t)
- Визуализация метрик отражает выходные сигналы Y(t)
### Связь с Практикой 5 (Требования):
- Каждый элемент GUI соответствует конкретным User Stories
- Технические требования реализованы в функциональности интерфейса
- Критерии приемки проверяются через демонстрационные сценарии
